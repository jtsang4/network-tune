# Network Tune

Agent-oriented TCP/BBR tuning for Debian and Ubuntu VPS hosts.

This repository packages a Codex skill and supporting scripts that let an agent tune a remote VPS over SSH. The workflow is functionally equivalent to Eric86777/vps-tcp-tune option `66`, with extra guardrails for agent execution: safer XanMod package selection, manual bandwidth profiles, staged reboot handling, persistence checks, and concise verification output.

## What It Does

- Installs a XanMod kernel for BBR v3 when the host supports custom kernels.
- Applies Eric86777/vps-tcp-tune's BBR direct/endpoint tuning profile.
- Configures `fq`, TCP buffers, MSS clamp, `initcwnd/initrwnd`, file limits, and swap guidance.
- Applies DNS purification through `systemd-resolved`.
- Applies Realm/conntrack timeout tuning.
- Optionally disables IPv6, matching the upstream option `66` default flow.
- Reboots and verifies persistence when requested.

## Quick Start

Run from the repository root or the skill directory:

```bash
skills/network-tune/scripts/agent-tcp-tune.sh \
  --target root@HOST \
  --bandwidth-mbps 1000 \
  --region asia \
  --reboot-verify
```

For a Singapore, Hong Kong, Japan, or Korea 1Gbps VPS, use:

```bash
--bandwidth-mbps 1000 --region asia
```

For a high-latency cross-ocean service path, use:

```bash
--region overseas
```

## CLI Options

```text
--target root@host          SSH target. Root access is expected.
--bandwidth-mbps N          Expected bandwidth used for tuning. Default: 1000.
--region asia|overseas      Buffer profile region. Default: asia.
--reboot-verify             Reboot after kernel install and after tuning, then verify.
--no-dns                    Skip DNS purification.
--no-realm                  Skip Realm/conntrack tuning.
--keep-ipv6                 Keep IPv6 enabled.
--ssh-option VALUE          Add one raw ssh option, repeatable.
```

## Agent Skill Usage

The core agent entrypoint is:

```text
skills/network-tune/SKILL.md
```

An agent should use the skill when a user asks to optimize a remote VPS, tune BBR/TCP, install XanMod/BBR v3, reproduce `vps-tcp-tune` option `66`, diagnose why the upstream script returns to the menu, or verify post-reboot persistence.

The skill instructs the agent to prefer explicit bandwidth input over a single automatic Speedtest result. This matters because an unlucky Speedtest server can under-report a 1Gbps VPS and select a too-small TCP buffer.

## Remote Workflow

The local wrapper uploads these files to the remote host:

```text
/root/agent-tcp-tune/remote-tcp-tune.sh
/root/agent-tcp-tune/net-tcp-tune.sh
```

The remote script then runs:

1. `preflight`
2. `install-kernel`
3. optional reboot
4. `tune-66`
5. optional reboot
6. `verify`

Remote logs are written under:

```text
/root/agent-tcp-tune/logs
```

## Verification

A successful final verification should show values like:

```text
KERNEL=6.x-xanmod
CONGESTION=bbr
QDISC=fq
TCP_RMEM=4096 87380 16777216
TCP_WMEM=4096 65536 16777216
BBR_PERSIST_ENABLED=enabled
DNS_PERSIST_ENABLED=enabled
```

For a 1Gbps Asia profile, Eric's policy maps to a 16MB TCP buffer.

## Safety Notes

This project performs root-level network and kernel changes on remote machines. Use it on VPS hosts where reboot and network configuration changes are acceptable.

Key safety defaults:

- Create backups under `/root/agent-tcp-tune/backup-*`.
- Keep DNS rollback scripts generated by the upstream DNS purification module.
- Remove temporary XanMod APT sources after kernel installation.
- Verify persistence after reboot when the user permits it.
